# Formal Complaint Documentation – Disturbance Handling Case

## Overview
This repository documents a formal complaint submitted to a residential property management company regarding the handling of repeated disturbance reports in a rental property.

The purpose of this repository is to:
- Provide transparent documentation of communication sent to higher-level management
- Preserve factual records related to published disturbance-handling procedures
- Demonstrate alignment between reported experiences and publicly stated policies
- Ensure clarity, traceability, and accountability in escalation processes

This repository is intended for informational and documentation purposes only.

---

## Scope and Intent
The materials contained here focus on **process, procedure, and communication**, not on individuals.

Specifically, this repository:
- Does **not** name or identify individual employees
- Does **not** allege criminal conduct
- Does **not** contain personal data beyond what is necessary for contextual understanding
- Describes events and impacts from the perspective of the author, based on direct experience and written correspondence

The intent is to document concerns regarding procedural handling of disturbance reports, particularly in relation to publicly published guidance and applicable tenancy legislation.

## Repository Contents

### 1. `email_to_higher_level_manager.md`
This file contains the full text of a formal escalation email addressed to higher-level management.

The email:
- Summarizes a timeline of reported disturbances
- References publicly available disturbance-handling procedures
- Cites relevant sections of Finnish tenancy law (*Laki asuinhuoneiston vuokrauksesta*, AHVL 481/1995, including §§ 61–62)
- Requests clarification on concrete actions taken
- Is written in a professional, factual, and non-accusatory manner

This document represents the primary narrative and formal complaint.
